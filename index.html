Practice Before the Patient

This project develops a simulation-based software platform that enables educators to create customizable patient scenarios and deliver immediate, targeted feedback on student performance. The platform supports athletic training students by strengthening clinical reasoning, confidence, and readiness for real-world patient care through repeated, low-stakes practice.

Final Assessment

The current authentication setup is development-only and was never finished for production use. The application does not currently use university SSO or a real login flow. Instead, it relies on a temporary local access model for testing and demos, where user roles are simulated inside the app. This was acceptable for development, but it is not appropriate for a production deployment and would need to be replaced with university credential-based sign-in.

Originally, the plan was different. The team first discussed hosting with an on-premises database, but after internal discussions OIT decided that Azure would be the better long-term approach. OIT's reasoning was that the project is already tied to services and components that fit better in Azure, including the future SSO app registration, Docker hosting, and the future Azure OpenAI service.

They also noted that keeping the app in Azure would simplify networking and production support, since connecting an on-premises database to Azure resources would require extra firewall configuration and add maintenance complexity. Their recommendation was to migrate the application stack into Azure, use an Azure app for SSO, use managed identity for secure access to the database and OpenAI service without storing passwords or API keys in code, and ideally use Bicep to provision the environment.
